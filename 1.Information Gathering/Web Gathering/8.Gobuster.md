# Gobuster

## Order of operations
```bash
1. Directory enumeration    →  find valid paths/folders
2. File enumeration         →  find files within those dirs (.php, .html, .txt, etc.)
3. Parameter enumeration    →  find parameters on interesting files
4. Value fuzzing            →  fuzz the parameter values (LFI lists, SQLi, etc.)
```

## Parameter Enumeration

```bash
# Once you have the directories and files, and discovered something like index.php, now you can fuzz parameters. Example:
#index.php being the discovered file
http://192.168.215.16/meteor/index.php 

# Fuzz Command
#NOTE: This part is highly dependent on a good wordlist. You may get ZERO results dependent on what wordlist you use.

ffuf -w /usr/share/seclists/Discovery/Web-Content/burp-parameter-names.txt -u 'http://192.168.215.16/meteor/index.php?FUZZ=test' -fs <baseline_size>

# Larger Wordlist
ffuf -w /usr/share/seclists/Discovery/Web-Content/raft-large-words.txt -u 'http://192.168.215.16/meteor/index.php?FUZZ=test' -fs 7046

# If you got results, proceed to directory traversal. If not, do a manual search through "View Source".
# Ctrl - F -> ? 
# Or -> index.php?
```
