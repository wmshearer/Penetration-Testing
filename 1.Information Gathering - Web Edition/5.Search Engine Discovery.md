## Search Engine Discovery (OSINT)

### Summary
Search engine discovery uses public search engines and advanced operators (“Google dorks”) to uncover exposed pages, files, credentials, and infrastructure-related data without directly interacting with the target.

------------------------------------------------------------

## EXERCISE: Search Operators Basics

### Task 1: Limit results to a specific domain

1. Search: `site:example.com`
   - Target-agnostic: `site:<DOMAIN>`
   - Example result:
     - Public pages indexed on `<DOMAIN>`

------------------------------------------------------------

### Task 2: Find login or admin pages

1. Search: `site:example.com inurl:login`
   - Target-agnostic: `site:<DOMAIN> inurl:login`
   - Example result:
     - `https://<DOMAIN>/login`
     - `https://<DOMAIN>/user/login`

2. Search: `site:example.com (inurl:login OR inurl:admin)`
   - Target-agnostic: `site:<DOMAIN> (inurl:login OR inurl:admin)`
   - Example result:
     - `https://<DOMAIN>/admin`
     - `https://<DOMAIN>/admin/login.php`

------------------------------------------------------------

### Task 3: Identify exposed documents

1. Search: `site:example.com filetype:pdf`
   - Target-agnostic: `site:<DOMAIN> filetype:pdf`
   - Example result:
     - `employee-handbook.pdf`
     - `internal-policy.pdf`

2. Search: `site:example.com (filetype:xls OR filetype:docx)`
   - Target-agnostic: `site:<DOMAIN> (filetype:xls OR filetype:docx)`
   - Example result:
     - `financials.xlsx`
     - `credentials.docx`

------------------------------------------------------------

## EXERCISE: Advanced Search Operators

### Task 1: Search page titles and body content

1. Search: `intitle:"confidential report"`
   - Target-agnostic: `intitle:"<PHRASE>"`
   - Example result:
     - Pages titled “Confidential Report”

2. Search: `intext:"password reset"`
   - Target-agnostic: `intext:"<KEYWORD>"`
   - Example result:
     - Pages referencing password reset workflows

------------------------------------------------------------

### Task 2: Locate configuration and backup files

1. Search: `site:example.com inurl:config.php`
   - Target-agnostic: `site:<DOMAIN> inurl:<CONFIG_FILE>`
   - Example result:
     - `https://<DOMAIN>/config.php`

2. Search: `site:example.com (ext:conf OR ext:cnf)`
   - Target-agnostic: `site:<DOMAIN> (ext:<EXT1> OR ext:<EXT2>)`
   - Example result:
     - `database.conf`
     - `app.cnf`

3. Search: `site:example.com filetype:sql`
   - Target-agnostic: `site:<DOMAIN> filetype:sql`
   - Example result:
     - `backup.sql`
     - `db_dump.sql`

------------------------------------------------------------

## EXERCISE: Combining Operators (Google Dorking)

### Task 1: Narrow or broaden results

1. Search: `site:example.com AND (inurl:admin OR inurl:login)`
   - Target-agnostic: `site:<DOMAIN> AND (inurl:<TERM1> OR inurl:<TERM2>)`
   - Example result:
     - Admin or login pages only on `<DOMAIN>`

2. Search: `site:bank.com NOT inurl:login`
   - Target-agnostic: `site:<DOMAIN> NOT inurl:<EXCLUDED_TERM>`
   - Example result:
     - Non-login pages on `<DOMAIN>`

------------------------------------------------------------

### Task 2: Exact phrases and wildcards

1. Search: `"information security policy"`
   - Target-agnostic: `"<EXACT_PHRASE>"`
   - Example result:
     - Policy documents containing the exact phrase

2. Search: `site:socialnetwork.com filetype:pdf user* manual`
   - Target-agnostic: `site:<DOMAIN> filetype:<FILETYPE> <KEYWORD>* <KEYWORD>`
   - Example result:
     - `user-guide.pdf`
     - `user-handbook.pdf`

------------------------------------------------------------

### Key Takeaway
- Search engine discovery is passive, legal OSINT.
- Combining operators exposes hidden pages, files, and misconfigurations.
- Best used early to guide deeper, active reconnaissance.
